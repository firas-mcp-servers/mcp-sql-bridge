# Future ideas (roadmap beyond 0.2.x)

This document tracks potential enhancements. None are committed; they are ideas for discussion and prioritization.

- **New backends** — e.g. SQL Server, DuckDB, or other SQL dialects via optional drivers.
- **Advanced prompts** — more MCP prompts (e.g. “Generate a report from this schema”, “Find unused tables”).
- **Deeper analytics** — richer schema summary (row counts, indexes, FK graph) or query analysis.
- **Performance tuning** — connection pooling, configurable timeouts, or streaming for large results.
- **MCP SDK alignment** — adopt new MCP features (e.g. progress, streaming) as the protocol evolves.

Contribute ideas via [GitHub Discussions](https://github.com/firas-mcp-servers/mcp-sql-bridge/discussions) or [issues](https://github.com/firas-mcp-servers/mcp-sql-bridge/issues).
