# Launch plan and marketing

This document outlines a simple launch and marketing approach for MCP SQL Bridge.

## Goals

- Increase visibility among developers using MCP (Cursor, Claude Desktop, etc.).
- Drive adoption of the free SQLite workflow and optional Pro (Postgres/MySQL) usage.
- Support future paid offerings if we introduce them (e.g. Gumroad).

## Channels

| Channel        | Action                                                                 |
|----------------|------------------------------------------------------------------------|
| GitHub         | Clear README, topics (e.g. `mcp`, `sqlite`, `cursor`), release notes. |
| MCP ecosystem  | List in [MCP registry](https://github.com/modelcontextprotocol/servers) or community lists. |
| Product Hunt   | Optional one-time launch post when reaching a stable release (e.g. 1.0). |
| X / Twitter    | Short posts linking to repo, docs, and “how to use with Cursor/Claude”. |
| Discord/forums | Share in MCP or Cursor/Claude communities when relevant.               |

## Messaging

- **One line:** “Give your AI read-only access to your SQL databases (SQLite, Postgres, MySQL) over MCP.”
- **Key points:** Schema-aware, README context, read-only, stdio, no cloud, works in Cursor and Claude Desktop.

## Launch checklist

- [ ] README and docs up to date.
- [ ] At least one tagged release (e.g. `v0.2.0` or `v1.0.0`).
- [ ] CHANGELOG updated for the release.
- [ ] Repo description and topics set on GitHub.
- [ ] Optional: add to MCP registry or community list.
- [ ] Optional: announce in chosen channels (X, Discord, Product Hunt).

## Assets

- **Repo:** [github.com/firas-mcp-servers/mcp-sql-bridge](https://github.com/firas-mcp-servers/mcp-sql-bridge)
- **Docs:** This `docs/` folder and the main README.
- **Gumroad:** Use `GUMROAD_DESCRIPTION.txt` in the repo if listing a paid product later.
